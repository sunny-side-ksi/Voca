- 2섹션 × 20문항, 섹션당 35분

- 온스크린 계산기 사용 가능

## 문제 유형 1: Quantitative Comparison (양 비교)

### 형식
- Quantity A와 Quantity B를 비교하여 4개 중 하나 선택
  - A. Quantity A is greater.
  - B. Quantity B is greater.
  - C. The two quantities are equal.
  - D. The relationship cannot be determined from the information given.
- 4개 선택지는 **모든 QC 문제에서 동일**
- 두 수량을 명확히 계산할 수 있는 경우 **D 절대 선택 금지**

### 풀이 전략
1. **불필요한 계산 피하기** — 비교에 필요한 만큼만 계산
2. **도형은 축척대로 그려진 것이 아님** — 가능한 경우들을 검토
3. **숫자 대입 (Plug in)** — 0, 양수, 음수, 분수, 큰 수 등 다양하게 대입
   - A>B인 경우와 B>A인 경우 모두 나오면 → **D**
4. **비교 단순화** — 양변에 동일 연산 적용하여 단순화
   - `A ? B` → 양변에 더하기/빼기/양의 수 곱하기/나누기 가능
   - **음수 곱하기·나누기 시 부등호 방향 역전**

### 예시 풀이
```
Quantity A: x² + 1
Quantity B: 2x − 1

단순화: x² + 1 ? 2x − 1
       x² − 2x + 2 ? 0
       (x−1)² + 1 ? 0
       → (x−1)² ≥ 0이므로 항상 > 0 → A > B
정답: A
```

## 문제 유형 2: Multiple Choice — Select ONE Answer (단일 선택 5지선다)

### 형식
- 5개 선택지(원형 버튼) 중 정확히 1개 선택

### 전략
1. **정답은 반드시 선택지 안에** — 없으면 계산 오류
2. **선택지를 먼저 훑어보기** — 어느 정도 정밀도가 필요한지 파악
3. **역추산(Backsolving)** — 선택지를 역으로 대입
4. **어림값(Estimation)** — 정확한 계산이 불필요할 때

---

## 문제 유형 3: Multiple Choice — Select ONE OR MORE Answers (복수 선택)

### 형식
- 선택지 옆에 **정사각형 체크박스(□)**
- 1개 이상 정답, 해당하는 것 **모두** 선택
- **부분 점수 없음** — 전부 맞아야 점수 인정

### 전략
1. 개수 명시 여부 확인 (명시 없으면 전부 가능)
2. **각 선택지를 독립적으로 평가**
3. 수치 선택지에서 최솟값·최댓값 먼저 파악
4. 패턴 인식 활용 (예: 거듭제곱의 일의 자릿수 주기)

---

## 문제 유형 4: Numeric Entry (숫자 직접 입력)

### 형식
- 정수 또는 소수 → 단일 입력창
- 분수 → 분자·분모 별도 입력
- 기약분수로 줄이지 않아도 됨
- Transfer Display 버튼으로 계산기 값 이전 가능

### 전략
1. **단위 확인** (문항에서 요구하는 단위)
2. **반올림 지시 엄격히 따를 것** — 중간 계산은 정확하게, 최종에서만 반올림
3. **Transfer Display 주의** — 퍼센트/반올림 필요 시 수동 수정 필수
4. 합리성 검토로 교차 확인

## 일반 문제풀이 단계 (ETS 권고 3단계)

**Step 1: 문제 이해** — 주어진 정보 / 조건 / 구하는 것 파악

**Step 2: 전략 실행** — 적절한 전략 선택 및 실행, 막히면 전략 변경

**Step 3: 답 검증** — 질문이 요구한 것을 답했는가 / 합리적인가 / 계산 오류 없는가

## Calculator 사용 팁

### 기본 원칙
- **필요 없으면 쓰지 마라** — 간단한 계산은 암산이 더 빠름
- 어려운 계산(장나눗셈, 제곱근, 자릿수 큰 수)에 활용
- **분수 답이 필요하면 소수 개입 자제**
- 먼저 어림값으로 범위 파악 → 입력 오류 방지

### 연산 순서 (GRE 온스크린 계산기)
```
괄호 → 지수(√ 포함) → 곱셈·나눗셈(좌→우) → 덧셈·뺄셈(좌→우)
```
- **일반 보급형 계산기와 다름**: `1 + 2 × 4 = 9` (GRE 계산기), `= 12` (일반 계산기)

### 주요 기능
- MR (메모리 호출) / MC (메모리 초기화) / M+ (메모리에 더하기)
- ± 버튼: 양수/음수 전환
- Transfer Display: Numeric Entry 문항에서 계산기 값을 입력창으로 이전
- 8자리까지 표시 (초과 시 ERROR)
- 0으로 나누기, 음수의 제곱근 → ERROR

## 수학적 관례 (Mathematical Conventions)

### 수와 기호
- 모든 수는 **실수**; 허수 고려 안 함
- `1 billion = 10⁹` (일부 국가 표기와 다름)
- **반올림**: 정중앙(halfway)이면 양수는 큰 쪽으로, 음수는 절댓값이 큰 쪽으로
  - 예: 23.5 → 24, −36.5 → −37
- 약수/인수: rs = t이면 r, s는 t의 인수 (음의 정수도 인수 포함)
- 나머지: n = qd + r, **0 ≤ r < |d|**
  - 예: −17 ÷ 7 → 몫 = −3, 나머지 = 4

### 연산 순서
```
괄호 → 지수 → 부호역전(negation) → 곱셈·나눗셈(좌→우) → 덧셈·뺄셈(좌→우)
```
- **중요**: −3² = −9 (지수가 부호역전보다 우선)
- (−3)² = 9 (괄호가 우선)

### 기하 도형 해석 규칙
- **도형은 축척대로 그려진 것이 아님** (좌표계·그래프 제외)
- 시각적 추정 금지
- 좌표계(xy-plane, 수직선), 막대그래프, 원그래프, 꺾은선 그래프 → 축척대로 → 시각적 읽기 가능
